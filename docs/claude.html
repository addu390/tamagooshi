    Quickstart

    The Claude brand ships in the repo as brands/claude/: Clawd as the mascot pack, the built-in terra terracotta theme plus a custom clay dark variant, the Claude agent enabled, and BLE as the only transport. Try it in the simulator first, no board needed.

    Simulator

    The same firmware, drawn in a desktop window.

    make sim BRAND=claude

    Pair with Claude

    One command on the machine that runs Claude Code.

    pip install -e hub   # once, Python 3.10+
make hub             # pairs over BLE

    Once paired, hold the side button to talk to Claude, watch Clawd react as it works, and approve or deny actions from the device.

    Configuration

    This builder is pre-seeded with the Claude brand. Adjust anything, then copy or download the config.yaml into brands/claude/.

    The stock Claude config keeps it lean: the Clawd pack, the terracotta themes, the Claude agent, and BLE only. Anything you leave out stays out of the image.

    Build & flash

    Flash straight from the browser over USB. Your config above rides along as a small partition, so the device boots as the Claude edition.

    Build a lean image locally

    For the smallest image, bake the brand in at compile time instead.

    make brand BRAND=claude                              # generate headers into firmware/.gen/current
cd firmware && pio run -e m5stickc-plus -t upload    # build + flash (pick your board's env)
